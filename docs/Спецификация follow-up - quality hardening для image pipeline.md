# Спецификация follow-up: quality hardening для image pipeline

## 0. Назначение документа
Этот документ фиксирует следующий этап после закрытия базового Level 1 guardrail для `image semantic-redraw-mode`. 
Его цель — внедрить индустриальные стандарты обработки изображений, решить фундаментальные ограничения текущих генеративных моделей (DALL-E 3) и описать пакет доработок, который улучшает качество эвристик и устойчивость image pipeline на реальных кейсах.

---

## 1. Scope follow-up этапа
В этот этап входят:
1. внедрение **Smart Routing (Умный Bypass)** на этапе `image_analysis` для сложных текстовых инфографик;
2. архитектурный отказ от `images.edit` в пользу двухшагового `semantic_redraw_structured` (Vision Extractor + Generate);
3. интеграция альтернативного рендера (Mermaid.js) для высоконагруженных текстовых схем;
4. расширение UI / run-log для прозрачной маршрутизации.

---

## 2. Архитектурные изменения (The Core Problem Solvers)

### 2.1. Отказ от images.edit (Semantic Redraw Direct)
**Проблема:** Эндпоинт `images.edit` (даже с gpt-image-1) плохо рендерит текст и искажает структуру схем, а также часто падает из-за отсутствия альфа-канала (прозрачности) в исходниках (JPEG).
**Решение:** Полный переход на стратегию `semantic_redraw_structured`.
*   **Шаг 1:** Модель Vision (`gpt-4o`) анализирует исходную диаграмму и генерирует исчерпывающее текстовое описание её структуры.
*   **Шаг 2:** Эндпоинт `images.generate` с моделью `dall-e-3` получает это описание и рисует новую инфографику с нуля.

### 2.2. Smart Routing и Умный Bypass
**Проблема:** DALL-E 3 галлюцинирует при попытке отрендерить инфографику с высокой плотностью текста (таблицы, схемы на 15+ узлов).
**Решение:** На этапе `analyze_image` добавляется классификатор:
*   `simple_diagram` (до 7 слов/узлов) -> отправляется в DALL-E 3.
*   `dense_text` / `complex_diagram` -> активирует Bypass (сохранение оригинала) ИЛИ переходит на векторную кодовую генерацию (Mermaid).

## 3. Детализация доработок по модулям

### 3.1. Улучшение `image_analysis.py`
*   Добавить промпт-классификатор, возвращающий JSON с полями: `complexity_score` (1-10), `node_count` (int), `recommended_strategy` (enum: dall-e-3, bypass, mermaid).
*   При `node_count` > 7 принудительно выставлять Bypass.

### 3.2. Улучшение `image_generation.py`
*   Удалить фолбэки на `dall-e-2`. 
*   Добавить обработчик генерации Mermaid-кода для сложных текстовых схем (вместо DALL-E 3).

## 4. Практические примеры реализации (Best Practices)

В этом разделе собраны проверенные промпты и структуры данных для внедрения в код.

### 4.1. Промпт для Smart Routing (Шаг 1: Анализ)
Этот системный промпт должен использоваться в `gpt-4o` (Vision) для определения судьбы картинки:

```text
Ты — эксперт по анализу документов. Оцени приложенное изображение.
Твоя задача — определить, сможет ли диффузионная модель (DALL-E 3) качественно перерисовать это изображение, сохранив ВЕСЬ текст без искажений.

Правила:
1. Если на картинке больше 7 логических блоков с текстом, или это плотная таблица — модель с этим НЕ справится.
2. Если картинка содержит сложную непрямоугольную метафору (например, клубок ниток, дерево с корнями) с множеством подписей — модель с этим НЕ справится.

Ответь строго в формате JSON:
{
  "image_type": "photo" | "simple_diagram" | "dense_diagram",
  "text_node_count": <int>,
  "can_dalle_reproduce_exactly": <bool>,
  "recommended_route": "dalle_3" | "mermaid_code" | "bypass"
}
```

### 4.2. Промпт для извлечения структуры (Шаг 2: Если route == dalle_3)
Если картинка простая, извлекаем её для DALL-E 3:

```text
Опиши эту диаграмму так, чтобы генератор изображений смог нарисовать её с нуля.
КРИТИЧЕСКИ ВАЖНО:
1. Выпиши ДОСЛОВНО весь текст, который есть на картинке, сохраняя язык оригинала (русский/украинский/английский).
2. Опиши визуальный стиль (цвета, геометрические формы).
3. Опиши связи (какой блок куда ведет).
Сформируй итоговый текст как готовый Prompt для генератора.
```

### 4.3. Решение для плотных схем: Генерация кода (Если route == mermaid_code)
Если картинка сложная (как пример с "Факты vs Манипуляции"), мы НЕ отправляем её в DALL-E 3. Мы просим `gpt-4o` перевести её в код:

```text
Переведи эту сложную инфографику в синтаксис Mermaid.js (flowchart TD или mindmap).
Сохрани ВСЕ текстовые узлы дословно на языке оригинала. 
Используй стилизацию узлов (classDef), чтобы передать цвета (например, зеленый для фактов, красный для манипуляций).
Не придумывай ничего от себя, только конвертация графики в код.
```
*Интеграция в проект:* Полученный Mermaid-код можно либо вставлять в DOCX как текст, либо рендерить через headless-браузер (или API `mermaid.ink`) в чистую PNG-картинку и вставлять обратно в документ. Это гарантирует 100% читаемость текста и 0% галлюцинаций.

## 5. Порядок внедрения

1. **Routing & Bypass:** Внедрить JSON-классификатор (п. 4.1) в `image_analysis.py`. Настроить отбраковку (`bypass`) сложных схем.
2. **Two-Step DALL-E:** Обновить `image_generation.py`, используя промпт из п. 4.2.
3. **Advanced Rendering (Опционально):** Добавить обработку `mermaid_code` для тех случаев, когда сложную схему все-таки необходимо перевести или изменить.
