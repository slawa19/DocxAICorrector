# Режимы обработки изображений в UI

Этот документ описывает текущий pipeline для четырех опций в поле `Режим обработки изображений` в UI.

## 1. Просто улучшить без перерисовки

UI-значение: `safe`

Когда использовать:
- фотографии;
- скриншоты;
- изображения, где нельзя рисковать потерей текста или структуры;
- любые случаи, когда нужно минимальное вмешательство.

Как работает pipeline:
- `image_analysis.py` все равно анализирует изображение, чтобы зафиксировать тип и маршрут;
- `image_generation.py` вызывает локальный `safe`-path без генеративной перерисовки;
- Pillow делает мягкое улучшение: autocontrast, немного contrast/color/sharpness;
- layout, пиксельная структура и текст не пересобираются заново;
- Level 1 post-check не нужен, потому что semantic redraw не выполняется.

Что дает:
- минимальный риск галлюцинаций;
- почти полное сохранение исходного содержания;
- ограниченное визуальное улучшение.

Ограничения:
- не делает новую композицию;
- не исправляет слабый дизайн исходника;
- не превращает таблицу/инфографику в более красивый editorial-style layout.

## 2. Свободная AI-перерисовка по смыслу

UI-значение: `semantic_redraw_direct`

Когда использовать:
- инфографика;
- визуальные explainers;
- слайды, постеры, схемы, где важна красота и композиция;
- случаи, когда нужен более творческий результат, а не буквальная реконструкция координат.

Как работает pipeline:
- `image_analysis.py` определяет тип изображения и извлекает метаданные;
- `image_generation.py` запускает creative hybrid path;
- сначала `Responses API` делает `creative redraw brief`: смысл, hierarchy, читаемый текст, группировки, возможности для более сильной композиции;
- затем `Images.generate` генерирует новый visual с нуля по этому brief;
- prompt специально разрешает менять композицию, spacing, typography, card shapes, connector styling и color hierarchy;
- prompt отдельно запрещает spreadsheet-like / Excel-like результат;
- если creative generate падает, pipeline пытается сделать direct edit fallback;
- если и он не проходит, включается structured fallback.

Что дает:
- лучший шанс получить красивый, современный, editorial-like результат;
- больше свободы в цветах, шрифтах, визуальном ритме и композиции;
- лучше подходит для картинок вроде `pic1`, где буквальная таблица выглядит слишком механически.

Ограничения:
- это самый рискованный режим по точности;
- модель может лучше стилизовать композицию, но хуже удерживать буквальную структуру, чем exact-mode;
- поэтому после redraw идет post-check и при проблемах возможен fallback на safe/original.

## 3. Точная AI-перерисовка схем и таблиц

UI-значение: `semantic_redraw_structured`

Когда использовать:
- таблицы;
- блок-схемы;
- диаграммы с важными связями и стрелками;
- графики и структурные изображения, где важнее сохранить контент и отношения, чем сделать свободный redesign.

Как работает pipeline:
- `image_analysis.py` выбирает conservative structured route;
- `image_generation.py` использует Vision + `Images.generate` с conservative office-presentation-style prompt;
- `Responses API` извлекает layout/spec и список читаемых текстовых элементов;
- prompt требует сохранить контент, чтение, связи, ячейки и блоки, но разрешает улучшить typography, spacing, alignment и visual hierarchy;
- deterministic reconstruction остается fallback-путем, а не целевым пользовательским результатом;
- после этого `image_validation.py` делает Level 1 post-check: semantic match, text match, structure match, image type consistency;
- если redraw не проходит validation, pipeline возвращается к safe/original.

Что дает:
- лучший контроль над сохранением структуры и содержания;
- меньше риск потерять стрелки, ячейки, блоки и читаемый текст;
- предсказуемее для технических схем и таблиц;
- более аккуратный результат в стиле Office presentation, а не буквальную пиксельную копию исходника.

Ограничения:
- визуально обычно сдержаннее, чем free-mode;
- если исходник очень грязный или текст слишком плотный, structured-mode все равно может откатиться в safe/original;
- это не режим свободного редизайна, а режим консервативного улучшения структуры.

## 4. Сгенерировать 3 варианта для выбора

UI-значение: `compare_all`

Когда использовать:
- когда заранее не ясно, какой режим даст лучший результат для конкретной картинки;
- когда нужно сравнить safe, creative и conservative варианты бок о бок;
- когда пользователь хочет сам выбрать итоговый вариант перед финальной сборкой DOCX.

Как работает pipeline:
- `image_pipeline.py` для каждого изображения сразу готовит три варианта: `safe`, `semantic_redraw_direct`, `semantic_redraw_structured`;
- варианты сохраняются в `comparison_variants` внутри `ImageAsset`;
- UI показывает оригинал и три подготовленных результата рядом;
- пользователь выбирает вариант для каждого изображения отдельно;
- после этого приложение пересобирает итоговый DOCX и вставляет именно выбранные изображения.

Что дает:
- позволяет сравнивать три подхода на одном и том же изображении;
- убирает необходимость угадывать правильный режим заранее;
- делает финальный выбор управляемым на уровне пользователя, а не только автоматического роутинга.

Ограничения:
- дороже и медленнее, чем один выбранный режим;
- для некоторых изображений semantic-варианты могут быть недоступны и тогда останется только safe/original;
- итоговый DOCX обновляется после явного применения выбранных вариантов.

## Как выбирается итоговый путь внутри semantic-режимов

Pipeline устроен в два слоя:

1. Пользователь выбирает режим в UI.
2. Внутри режима анализ изображения уточняет допустимый маршрут и fallback-правила.

Примеры:
- если пользователь выбрал `safe`, semantic redraw не запускается вообще;
- если выбран `semantic_redraw_direct`, режим больше не должен насильно проваливаться в deterministic reconstruction только потому, что изображение структурное;
- если выбран `semantic_redraw_structured`, основной путь теперь generation-first с conservative office-style prompt, а deterministic reconstruction остается fallback-механизмом;
- если выбран `compare_all`, pipeline готовит safe, free и structured варианты сразу и ждет ручного выбора пользователя;
- если Vision видит слишком плотный нелатинский текст или слишком высокую текстовую плотность, semantic route может быть принудительно downgraded в `safe_mode`.

## Практическое правило выбора

- Нужна максимальная сохранность: выбирайте `Просто улучшить без перерисовки`.
- Нужна красота, композиция, современный redesign: выбирайте `Свободная AI-перерисовка по смыслу`.
- Нужна консервативная, но более аккуратная схема с лучшей типографикой и компоновкой: выбирайте `Точная AI-перерисовка схем и таблиц`.
- Нужен выбор между всеми тремя вариантами перед финальным DOCX: выбирайте `Сгенерировать 3 варианта для выбора`.
