# DocxAICorrector

MVP для литературного редактирования `.docx` через OpenAI с промежуточным Markdown и обратной сборкой в `.docx`.

## Что уже реализовано

- загрузка `.docx` через Streamlit;
- извлечение текста из абзацев со структурными эвристиками;
- сборка смысловых блоков из нескольких абзацев вместо обработки по одному абзацу;
- заголовки приклеиваются к следующему тексту, последовательные элементы списка держатся вместе;
- если абзац или блок длиннее лимита, он отправляется в обработку целиком отдельным блоком;
- для каждого блока автоматически добавляется соседний контекст до и после только для понимания смысла;
- последовательная отправка блоков в OpenAI Responses API;
- ограниченные retry для временных API-сбоев;
- журнал обработки по блокам прямо в интерфейсе;
- файловый лог приложения с `event_id` и stack trace для всех ошибок и ключевых этапов;
- сохранение промежуточного Markdown-результата при ошибке сборки или сбое API;
- сброс последнего результата без перезапуска приложения;
- сборка Markdown обратно в `.docx` через Pandoc;
- предпросмотр Markdown и скачивание итогового файла.

## Выбранные решения

- SDK: актуальный `openai` Python SDK;
- API: `client.responses.create(...)`;
- дефолтная конфигурация хранится в `config.toml`, локальные override можно задавать через `.env`;
- модели по умолчанию: `gpt-5.4`, `gpt-5.4-pro`, `gpt-5.2`, `gpt-5.1`, `gpt-5-mini`;
- оптимальный retry для MVP: `3` попытки только для временных ошибок (`429`, `5xx`, connection/timeout);
- системный промпт вынесен в `prompts/system_prompt.txt`;
- имя исходного файла и метаданные пока не сохраняются специально.

## Как теперь устроена обработка

Приложение больше не считает абзац минимальной единицей редактирования. Вместо этого оно строит смысловые блоки и передает в модель такую структуру:

- `target_block` — блок, который нужно реально отредактировать;
- `context_before` — предыдущий текст только для понимания смысла;
- `context_after` — следующий текст только для понимания смысла.

Правила сборки блоков в текущей реализации:

- заголовок образует начало нового блока и принудительно связывается хотя бы со следующим абзацем;
- последовательные пункты списка группируются вместе;
- обычные абзацы объединяются в блок до достижения рабочего лимита;
- слишком длинный абзац не режется внутри себя и уходит отдельным блоком;
- размер соседнего контекста вычисляется автоматически от `chunk_size`.

Это дает модели больше локального контекста, но при этом сохраняет управляемую сборку результата: на выходе она должна вернуть только исправленный `target_block`.

## Логи и ошибки

Теперь ошибки пишутся в двух видах:

- в интерфейсе показывается человекочитаемое сообщение с идентификатором вида `log: ...`;
- в файловом логе сохраняются технические детали, контекст и stack trace.

Основные файлы логов:

- `.run/app.log` — лог приложения Streamlit и вызовов OpenAI;
- `.run/project.log` — лог скриптов запуска и остановки проекта.

Пример: если в UI показано сообщение с `log: block_failed-...`, по этому идентификатору можно быстро найти полную запись в `.run/app.log`.

## Запуск в PowerShell

### 1. Создать и активировать виртуальное окружение

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Если PowerShell блокирует активацию, выполните:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
```

### 2. Установить зависимости Python

```powershell
pip install -r requirements.txt
```

### 3. Установить Pandoc

```powershell
winget install --id JohnMacFarlane.Pandoc -e
```

Проверьте установку:

```powershell
pandoc --version
```

### 4. Настроить ключ OpenAI

Создайте файл `.env` в корне проекта рядом с `app.py` и укажите в нем ключ:

```env
OPENAI_API_KEY=sk-...
```

Быстрый способ в PowerShell:

```powershell
Copy-Item .env.example .env
notepad .env
```

Приложение подхватывает ключ через `python-dotenv` автоматически при старте.

### 5. При необходимости отредактировать конфигурацию

Базовые значения лежат в `config.toml`:

```toml
default_model = "gpt-5-mini"
model_options = ["gpt-5.4", "gpt-5.4-pro", "gpt-5.2", "gpt-5.1", "gpt-5-mini"]
chunk_size = 6000
max_retries = 3
```

`chunk_size` теперь интерпретируется как целевой размер смыслового блока, а не как попытка жестко резать текст по абзацам.

Если нужно локально переопределить настройки без правки `config.toml`, добавьте в `.env`:

```env
DOCX_AI_DEFAULT_MODEL=gpt-5-mini
DOCX_AI_MODEL_OPTIONS=gpt-5.4,gpt-5.4-pro,gpt-5.2,gpt-5.1,gpt-5-mini
DOCX_AI_CHUNK_SIZE=6000
DOCX_AI_MAX_RETRIES=3
```

### 6. При необходимости отредактировать системный промпт

Правьте файл `prompts/system_prompt.txt`. Перезапустите Streamlit, чтобы приложение загрузило новую версию промпта.

### 7. Запустить приложение

```powershell
streamlit run app.py
```

Или через VS Code Run Task:

1. `Terminal -> Run Task`
2. Выберите `Project: Start`
Задачи запускаются через `pwsh`, чтобы совпадать с рабочим терминалом проекта в VS Code.

Для штатной остановки используйте задачу `Project: Stop`.

Что делает `Project: Start`:

- проверяет наличие `.venv`, `app.py`, `.env`;
- проверяет Python-зависимости и `pandoc`;
- валидирует `OPENAI_API_KEY`;
- запускает Streamlit в фоне;
- выводит итоговый статус, URL и пишет шаги запуска в `.run/project.log`.

Важно: задача `Project: Start` остается активной, пока сервер работает. Это сделано специально, чтобы VS Code Tasks не завершал дочерний процесс Streamlit сразу после старта.

Рабочие файлы запуска и логирования:

- PID: `.run/streamlit.pid`
- лог приложения: `.run/app.log`
- лог start/stop-скриптов: `.run/project.log`

### 8. Быстрый smoke-test

1. Подготовьте небольшой `.docx` на 2-3 абзаца без таблиц и картинок.
2. Запустите приложение и загрузите этот файл.
3. Оставьте модель `gpt-5-mini` и нажмите кнопку обработки.
4. Убедитесь, что появился журнал блоков, предпросмотр Markdown и кнопки скачивания.
5. Скачайте итоговый `.docx` и проверьте, что файл открывается в Word без ошибок.

Для проверки окружения до запуска UI достаточно убедиться, что:

```powershell
pandoc --version
python -c "from dotenv import load_dotenv; load_dotenv(); import os; print('OK' if os.getenv('OPENAI_API_KEY') else 'NO_KEY')"
```

## Ограничения MVP

- не сохраняются картинки, таблицы, сноски и сложная верстка Word;
- контекст ограничен соседними блоками, а не всей книгой целиком;
- большие книги будут обрабатываться заметно дольше.
